# 元素伤害系统集成说明

## 问题描述

之前在 `weapon_data.gd` 中定义的元素伤害属性（`fire_damage`, `ice_damage`, `lightning_damage`, `poison_damage`）是独立变量，与 stat 系统不匹配，无法通过 `StatsComponent` 进行统一管理。

## 解决方案

### 1. 扩展 StatType 枚举

在 [`StatModifier.StatType`](../scripts/stats/stat_modifier.gd:35-42) 中新增了 6 种元素伤害类型：

```gdscript
FIRE_DAMAGE,        ## 火焰伤害
ICE_DAMAGE,         ## 冰霜伤害
LIGHTNING_DAMAGE,   ## 闪电伤害
POISON_DAMAGE,      ## 毒素伤害
DARK_DAMAGE,        ## 暗影伤害
HOLY_DAMAGE,        ## 神圣伤害
```

### 2. 更新 StatsData

在 [`StatsData`](../scripts/stats/stats_data.gd:42-57) 中添加了对应的基础属性：

```gdscript
@export var fire_damage: float = 0.0
@export var ice_damage: float = 0.0
@export var lightning_damage: float = 0.0
@export var poison_damage: float = 0.0
@export var dark_damage: float = 0.0
@export var holy_damage: float = 0.0
```

### 3. 更新 StatsComponent

在 [`StatsComponent._get_base_value()`](../scripts/stats/stats_component.gd:385-390) 中添加了元素伤害的获取逻辑。

### 4. 重构 WeaponData

- **移除**了独立的元素伤害变量
- **改用** `stat_modifiers` 数组来添加元素伤害
- **更新**了 `get_damage_description()` 方法，从修正器中读取元素伤害

## 使用方法

### 方法 1: 在代码中创建武器

```gdscript
var weapon = WeaponData.new()
weapon.item_name = "火焰之剑"
weapon.min_physical_damage = 25.0
weapon.max_physical_damage = 35.0

# 添加火焰伤害修正器
var fire_mod = StatModifier.create_flat(
    StatModifier.StatType.FIRE_DAMAGE, 
    15.0,  # +15 火焰伤害
    "flame_sword"
)
weapon.stat_modifiers.append(fire_mod)
```

### 方法 2: 在 Godot 编辑器中配置

1. 创建或打开 `WeaponData` 资源
2. 在 **Stats Modifiers** 数组中添加新元素
3. 设置修正器属性：
   - `stat_type` → 选择对应的元素伤害类型（如 `FIRE_DAMAGE`）
   - `modifier_type` → 选择 `FLAT`（固定值）
   - `value` → 输入伤害数值
   - `source_id` → 输入武器ID（可选）

### 方法 3: 在 .tres 资源文件中配置

```tres
[sub_resource type="Resource" id="fire_damage_mod"]
script = ExtResource("stat_modifier_script")
stat_type = 37  # FIRE_DAMAGE 的枚举值
modifier_type = 0  # FLAT
value = 15.0
source_id = "flame_sword"

[resource]
script = ExtResource("weapon_data_script")
item_name = "火焰之剑"
stat_modifiers = Array[Resource]([SubResource("fire_damage_mod")])
```

## 优势

1. **统一管理**：元素伤害现在通过 `StatsComponent` 统一计算和缓存
2. **灵活性**：支持固定值、百分比、覆盖等多种修正类型
3. **可扩展**：轻松添加新的元素类型（如神圣、暗影等）
4. **兼容性**：与现有的装备系统完全兼容
5. **性能优化**：利用 `StatsComponent` 的缓存机制

## 战斗系统集成

在战斗中计算伤害时：

```gdscript
func calculate_attack_damage(attacker_stats: StatsComponent) -> Dictionary:
    return {
        "physical": attacker_stats.get_stat(StatModifier.StatType.PHYSICAL_DAMAGE),
        "magic": attacker_stats.get_stat(StatModifier.StatType.MAGIC_DAMAGE),
        "fire": attacker_stats.get_stat(StatModifier.StatType.FIRE_DAMAGE),
        "ice": attacker_stats.get_stat(StatModifier.StatType.ICE_DAMAGE),
        "lightning": attacker_stats.get_stat(StatModifier.StatType.LIGHTNING_DAMAGE),
        "poison": attacker_stats.get_stat(StatModifier.StatType.POISON_DAMAGE),
        "dark": attacker_stats.get_stat(StatModifier.StatType.DARK_DAMAGE),
        "holy": attacker_stats.get_stat(StatModifier.StatType.HOLY_DAMAGE),
    }
```

## 注意事项

1. 旧的武器资源文件需要移除 `fire_damage` 等属性
2. VSCode 可能会显示临时的语法错误，重启编辑器或重新加载项目即可解决
3. 元素伤害会受到目标的元素抗性影响（`RES_FIRE`, `RES_ICE` 等）
