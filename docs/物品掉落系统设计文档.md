## 系统概述

### 设计理念
针对 2D ARPG 游戏的掉落系统，参考《暗黑破坏神》、《流放之路》等经典 ARPG 的掉落机制：
- **数据驱动**: 使用 Resource 配置掉落表，无需修改代码
- **灵活配置**: 支持多种掉落模式和复杂条件
- **幸运系统**: 深度集成幸运值影响掉落
- **性能优化**: 对象池、批量生成等优化手段

### 系统架构

```
掉落系统
├── LootEntry (掉落条目) - 定义单个物品的掉落规则
├── LootTable (掉落表) - 组合多个掉落条目
├── LootGenerator (掉落生成器) - 生成掉落物到场景
├── DroppedItem (掉落物实体) - 场景中的可拾取物品
└── EnemyLootComponent (敌人掉落组件) - 敌人身上的掉落管理
```

### 核心特性
✅ **多种掉落模式** - 全部掉落、随机选择、加权随机  
✅ **条件掉落** - 等级限制、标签系统、概率控制  
✅ **幸运值影响** - 影响掉落率、数量、稀有度  
✅ **物品散开** - 圆形/随机散开，模拟真实掉落  
✅ **磁吸拾取** - 自动吸引到玩家  
✅ **首杀奖励** - 特殊掉落支持  
✅ **对象池** - 优化性能，减少实例化

## 核心组件

### 1. LootEntry (掉落条目)

**文件**: [`scripts/loot/loot_entry.gd`](scripts/loot/loot_entry.gd)

掉落表中的单个条目，定义一个物品的掉落规则。

#### 核心属性

```gdscript
## 物品配置
@export var item_data: ItemData           # 掉落的物品
@export var drop_chance: float = 1.0      # 掉落概率 (0.0-1.0)
@export var weight: int = 100             # 掉落权重

## 数量配置
enum QuantityMode { FIXED, RANDOM, WEIGHTED }
@export var quantity_mode: QuantityMode   # 数量模式
@export var min_quantity: int = 1         # 最小数量
@export var max_quantity: int = 1         # 最大数量

## 条件配置
@export var min_player_level: int = 1     # 最低等级需求
@export var max_player_level: int = 0     # 最高等级限制
@export var required_tags: Array[String]  # 必需标签
@export var excluded_tags: Array[String]  # 排除标签

## 幸运值影响
@export var luck_affects_chance: bool = true
@export var luck_affects_quantity: bool = false
```

#### 关键方法

```gdscript
# 计算最终掉落概率（考虑幸运值）
func get_final_drop_chance(luck_value: int) -> float

# 计算掉落数量（考虑幸运值）
func get_drop_quantity(luck_value: int = 0) -> int

# 检查条件是否满足
func check_conditions(player_level: int, context_tags: Array[String]) -> bool

# 检查是否触发掉落
func roll_drop(luck_value: int = 0) -> bool

# 创建物品实例
func create_item_instance(luck_value: int = 0) -> ItemInstance
```

**文件**: [`scripts/loot/loot_table.gd`](scripts/loot/loot_table.gd)

#### 掉落模式

```gdscript
enum DropMode {
    ALL,              # 所有条目都尝试掉落

    WEIGHTED_RANDOM   # 加权随机选择

```

```gdscript

@export var table_name: String
@export var drop_mode: DropMode = DropMode.ALL

## 掉落条目

## 数量配置（PICK_MULTIPLE 模式）
@export var min_picks: int = 1
@export var max_picks: int = 3
@export var allow_duplicates: bool = false

## 金币配置
@export var drops_gold: bool = true
@export var min_gold: int = 1
@export var max_gold: int = 10
@export var luck_affects_gold: bool = true
```

#### 关键方法

```gdscript
# 生成掉落物品（核心方法）
func generate_loot(
    player_level: int = 1,
    luck_value: int = 0,
    context_tags: Array[String] = []
) -> Dictionary  # 返回 {"items": Array, "gold": int}

# 获取所有可能掉落的物品（用于UI显示）
func get_all_possible_items() -> Array[ItemData]

# 获取平均掉落价值（用于平衡性测试）
```

## 最佳实践

### 1. 掉落表设计原则

#### 金字塔结构
```
传说 (1%)   ← 极稀有，高价值
  ↑
史诗 (5%)   ← 稀有，中高价值
  ↑
稀有 (15%)  ← 较常见，中等价值
  ↑
非凡 (30%)  ← 常见，低价值
  ↑
普通 (49%)  ← 基础掉落
```

#### 概率平衡
```gdscript
# ❌ 不好的设计 - 全是高概率
entry1.drop_chance = 0.8
entry2.drop_chance = 0.8
entry3.drop_chance = 0.8  # 玩家会拿到太多物品

# ✅ 好的设计 - 渐进概率
common_item.drop_chance = 0.6
uncommon_item.drop_chance = 0.3
rare_item.drop_chance = 0.1
epic_item.drop_chance = 0.05
legendary_item.drop_chance = 0.01
```

### 2. 性能优化

#### 使用对象池
```gdscript
# LootGenerator 配置
@export var use_pooling: bool = true
@export var pool_size: int = 50
```

#### 批量生成
```gdscript
# ❌ 不好 - 多次调用
for item in loot_items:
    loot_generator.spawn_item(item, position)

# ✅ 好 - 一次生成
var loot_data = {"items": loot_items, "gold": gold}
loot_generator.spawn_loot(loot_data, position)
```

#### 限制同屏掉落物数量
```gdscript
const MAX_DROPS_ON_SCREEN = 100

func spawn_loot(data, pos):
    var current_drops = loot_generator.get_all_drops()
    if current_drops.size() >= MAX_DROPS_ON_SCREEN:
        # 移除最旧的掉落物
        current_drops[0].despawn()
    
    loot_generator.spawn_loot(data, pos)
```

### 3. 平衡性测试

```gdscript
# 使用预览功能测试掉落价值
func test_loot_balance():
    var loot_table = load("res://data/loot_tables/boss_loot.tres")
    
    # 测试1000次，计算平均价值
    var avg_value = loot_table.get_average_value(10, 50)
    print("平均掉落价值: %d 金币" % avg_value)
    
    # 应该在合理范围内（例如：Boss价值应该是普通怪的10-20倍）
    assert(avg_value >= 100 and avg_value <= 500, "Boss掉落价值不合理")
```

### 4. 幸运值影响设计

```gdscript
# 建议的幸运值影响范围
# 每点幸运的影响应该是渐进的，避免过强

# 掉落概率：每点幸运 +0.5%
entry.luck_chance_scaling = 0.005

# 掉落数量：每点幸运 +1%
```

#### 破坏物掉落
```gdscript
class_name Destructible extends StaticBody2D

@export var drop_table: LootTable
@export var drop_chance: float = 0.5

func destroy():
    if randf() < drop_chance and drop_table:
        var loot_gen = get_node("/root/Main/LootGenerator")
        loot_gen.spawn_loot_from_table(
            drop_table,
            global_position,
            1,  # 破坏物不受等级影响
```
